## 程序如下
```
assume cs:code
code segment
start:  mov ax,0b800h
        mov es,ax
        mov ah,'a'
    s:  mov es:[160*12+40*2],ah
        inc ah
        cmp ah,'z'
        jna s
        mov ax,4c00h
        int 21h
code ends
end start
```

## 小结
这样有一个弊端，真正打包出来的exe运行速度是很快的。也就只能看清最后显示在屏幕上的z。
所以需要加一个延时的程序。
